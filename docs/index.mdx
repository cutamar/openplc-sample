This documentation describes a small PLC/ICS project based on **OpenPLC**, **Structured Text**, and **Modbus TCP**.

The goal of the project is to build and test a basic temperature control system. A simulated temperature value is sent to the PLC, and the PLC controls a cooling fan output based on defined temperature thresholds.

## Project Goal

The project demonstrates the following steps:

- installing and setting up OpenPLC Editor and Runtime
- writing a small Structured Text test program
- defining a temperature control scenario
- deploying the control logic to OpenPLC Runtime
- communicating with the PLC through Modbus TCP
- validating the PLC behavior with different simulated temperature values

## Control Scenario Summary

The control system uses a simulated temperature sensor as input and a cooling fan as output.

When the temperature becomes too high, the fan turns on. When the temperature drops back to a lower safe level, the fan turns off again.

This project uses two thresholds:

| Threshold | Value | Purpose |
|---|---:|---|
| Fan ON threshold | `86.0 °F` | Turns the fan on when the temperature is too high |
| Fan OFF threshold | `77.0 °F` | Turns the fan off once the system has cooled down |

Using two thresholds creates hysteresis and prevents the fan from rapidly switching on and off around a single temperature value.
