This page documents the validation tests and results for the OpenPLC temperature control project.

The purpose of this step is to confirm that the deployed PLC program reacts correctly to different simulated temperature values and that the cooling fan output behaves as expected.

## Validation Goal

The validation checks whether:

- the OpenPLC Runtime is running the deployed program
- simulated temperature values can be changed
- the PLC logic processes the updated temperature input
- the cooling fan output changes according to the defined thresholds
- the hysteresis behavior works as expected

## Control Logic Under Test

The PLC program uses the following Structured Text logic:

```pascal
IF Temperature_Sensor >= High_Threshold THEN
    Cooling_Fan := TRUE;
ELSIF Temperature_Sensor <= Low_Threshold THEN
    Cooling_Fan := FALSE;
END_IF;
```

The threshold values are:

| Variable | Value | Meaning |
|---|---:|---|
| `High_Threshold` | `86.0 °F` | Fan turns ON at or above this temperature |
| `Low_Threshold` | `77.0 °F` | Fan turns OFF at or below this temperature |

## Expected Hysteresis Behavior

The fan does not simply switch based on one threshold.

Instead:

- at or above `86.0 °F`, the fan turns ON
- at or below `77.0 °F`, the fan turns OFF
- between `77.0 °F` and `86.0 °F`, the previous fan state is kept

This means that a temperature such as `82.4 °F` can have two possible outcomes:

| Previous Fan State | `Temperature_Sensor` | Expected `Cooling_Fan` |
|---|---:|---|
| `FALSE` | `82.4 °F` | `FALSE` |
| `TRUE` | `82.4 °F` | `TRUE` |

This is expected because `82.4 °F` is between both thresholds.

## Test Method

The test method is:

1. Start OpenPLC Runtime.
2. Confirm that the temperature control program is running.
3. Change the simulated `Temperature_Sensor` value.
4. Observe the `Cooling_Fan` output.
5. Compare the observed result with the expected result.
6. Record the result as pass or fail.

The values can be changed manually in the OpenPLC debugger or through a Modbus TCP client.

## Observations

The expected behavior is that the fan does not rapidly switch on and off around a single threshold.

Instead, the fan uses hysteresis:

- it turns on at `86.0 °F`
- it turns off at `77.0 °F`
- it keeps its previous state between those values

This behavior is useful for temperature control because real temperature values may fluctuate slightly.

## Conclusion

The validation is successful if all observed fan states match the expected fan states.

The most important result is that the hysteresis behavior works correctly. The fan should stay on while the temperature cools down from the high threshold and should only turn off once the low threshold is reached.
