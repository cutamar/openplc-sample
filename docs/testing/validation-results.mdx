This page documents the extended validation of the OpenPLC temperature control project.

While the Modbus TCP client page already demonstrates the basic communication and a small set of functional tests, this page focuses on a more complete validation sequence. The goal is to test normal values, threshold boundary values, edge cases around the thresholds, and the hysteresis behavior of the cooling fan output.

## Control Logic Under Test

The PLC program uses two thresholds:

```pascal
IF Temperature_Sensor >= High_Threshold THEN
    Cooling_Fan := TRUE;
ELSIF Temperature_Sensor <= Low_Threshold THEN
    Cooling_Fan := FALSE;
END_IF;
```

| Variable | Value | Meaning |
|---|---:|---|
| `High_Threshold` | `86.0 °F` | Fan turns ON at or above this temperature |
| `Low_Threshold` | `77.0 °F` | Fan turns OFF at or below this temperature |

Between `77.0 °F` and `86.0 °F`, the fan keeps its previous state. This is the hysteresis behavior.

## Test Method

The validation was performed with a Python `pymodbus` script.

The script connects to OpenPLC Runtime over Modbus TCP, writes simulated values to `Temperature_Sensor`, reads the `Cooling_Fan` coil, and compares the observed output with the expected result.

The relevant mappings are:

| PLC Variable | PLC Location | Modbus Address | Access |
|---|---|---:|---|
| `Temperature_Sensor` | `%MD0` | `2048` | Write |
| `High_Threshold` | `%MD1` | `2050` | Read |
| `Low_Threshold` | `%MD2` | `2052` | Read |
| `Cooling_Fan` | `%QX0.0` | `0` | Read |

The `%MD` values are `REAL` values. Each `REAL` is encoded as two 16-bit Modbus holding registers.

## Validation Test Table

The following table describes the complete test sequence.

| Step | Input `Temperature_Sensor` | Expected `Cooling_Fan` | What This Validates |
|---:|---:|---|---|
| 1 | `68.0 °F` | `FALSE` | Normal low temperature, fan remains off |
| 2 | `76.9 °F` | `FALSE` | Just below low threshold, fan remains off |
| 3 | `77.0 °F` | `FALSE` | Exact low threshold, fan is off |
| 4 | `82.4 °F` | `FALSE` | Between thresholds before fan has turned on |
| 5 | `85.9 °F` | `FALSE` | Just below high threshold, fan remains off |
| 6 | `86.0 °F` | `TRUE` | Exact high threshold, fan turns on |
| 7 | `95.0 °F` | `TRUE` | Above high threshold, fan remains on |
| 8 | `86.1 °F` | `TRUE` | Just above high threshold, fan remains on |
| 9 | `82.4 °F` | `TRUE` | Between thresholds after fan has turned on, hysteresis keeps fan on |
| 10 | `77.1 °F` | `TRUE` | Just above low threshold, fan remains on |
| 11 | `77.0 °F` | `FALSE` | Exact low threshold, fan turns off |
| 12 | `68.0 °F` | `FALSE` | Back to low temperature, fan remains off |

This single sequence tests the low threshold, high threshold, values just below and above the thresholds, and the hysteresis region.
